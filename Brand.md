# Brand — Sources canoniques Frontguys

> **Hub Brand.** Cette section contient **toutes les sources qui définissent Frontguys** : identité, voix, visuel, composants. Source de vérité unique pour les LLM qui produisent des assets Frontguys.

## Quatre piliers

| Pilier | Contenu | Quand charger |
|--------|---------|---------------|
| **Identity** | Vision, mission, valeurs, positionnement, convictions, personas | Tout asset Frontguys |
| **Voice** | Tonalité, vocabulaire, règles d'écriture, exemples | Tout asset éditorial / textuel |
| **Visual** | Tokens, typographie, responsive, modes light/dark, illustrations | Tout asset visuel ou web |
| **Components** | 26 fiches composants Astro Frontguys | Asset web utilisant des composants *(Phase 2)* |

## Règle d'or


:::warning
**Ne jamais charger toute une section "au cas où".** Chaque sous-page indique précisément quoi charger en fonction de la tâche. Voir les tables "Quand charger quoi" dans les README de chaque pilier.

:::

## Sources canoniques externes

* **Identity** et **Voice** : la source de vérité est Knowtion (cette base de connaissance). Les pages ici sont la projection optimisée pour LLM.
* **Visual** : la source de vérité est Figma (tokens via TokensBrücke). Les valeurs réelles vivent dans `src/tokens/*.json` du repo Gitlab Frontguys.
* **Components** : la source de vérité est Gitlab `frontguys/accessibility-website`. Les fiches ici décrivent l'API et les règles d'usage.
