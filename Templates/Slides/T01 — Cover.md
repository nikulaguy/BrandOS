# Template T01 — Cover

## Rôle

Slide d'ouverture d'un deck. Pose le contexte (qui parle, à qui, sur quoi, à quelle date).

## Structure visuelle

```
┌─────────────────────────────────────────────┐
│                                             │
│   <Logo Frontguys (haut gauche)>           │
│                                             │
│   <Frontguys × Client>     (sur-titre)     │
│   <Titre principal du deck>    (H1 XL)     │
│   <Sous-titre / objet>         (subtitle)  │
│                                             │
│   <Visuel ou illustration de marque>       │
│                                             │
└─────────────────────────────────────────────┘
```

## Contenu type

| Élément | Format | Exemple (RAO BPI) |
|---------|--------|-------------------|
| Sur-titre | "Frontguys × {Client}" | "Bpifrance x Frontguys" |
| Titre principal (H1) | Titre du deck | "RAO - Global DD 2026" |
| Sous-titre | Sujet / objet | "Design & Engineering" |
| Date / réf | Optionnel, discret | —                 |

## Variables / paramètres

* **Logo Frontguys** : haut gauche, couleur full sur fond clair (mode dark : inverse)
* **Fond** : `semantic-background-default` (par défaut) ou `accent` pour deck à fort signal
* **Visuel optionnel** : illustration Frontguys (cf **Brand > Visual > Illustrations Index**)

## Règles de voix

* **Sur-titre** : factuel, "Frontguys × {Client}" — affirmation de la collaboration
* **Titre** : court, percutant (3-8 mots), pas de jargon
* **Sous-titre** : précise l'objet du deck en termes simples

## Anti-patterns

* ❌ Logo client trop gros (Frontguys s'efface)
* ❌ Logo client absent (rappel collaboration manqué)
* ❌ Titre vague ("Présentation", "Notre offre")
* ❌ Plus de 3 niveaux d'information (sur-titre + titre + sous-titre suffisent)
* ❌ Date dans le titre (la mettre en métadonnée discrète)
