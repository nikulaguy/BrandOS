# Template T06 — Four columns

## Rôle

Slide qui présente **4 axes parallèles d'un même concept**. À utiliser quand le concept se découpe naturellement en 4 (pas plus, pas moins). Très utilisé pour les **axes opérationnels** (Gouvernance, Coûts, Adoption, Sécurisation).

## Structure visuelle

```
┌─────────────────────────────────────────────┐
│   <Titre H2>                                │
│   <Sous-titre / accroche>                   │
│                                             │
│   ┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐       │
│   │ T1  │  │ T2  │  │ T3  │  │ T4  │       │
│   │Desc.│  │Desc.│  │Desc.│  │Desc.│       │
│   └─────┘  └─────┘  └─────┘  └─────┘       │
└─────────────────────────────────────────────┘
```

## Contenu type

### Exemple 1 — Axes opérationnels IA

| Gouvernance | Coûts | Adoption | Sécurisation |
|-------------|-------|----------|--------------|
| Nous étudions et avons comparé plusieurs dizaines de gateways IA (contrôle accès, usages et coûts liés aux LLM) pour assurer un déploiement maîtrisé. | Approche qui évite la dépendance d'un fournisseur unique. Test des LLM alternatifs (open-weight) et fournisseurs d'API serverless économiques. | Solutions adaptées aux profils d'utilisateurs, faciles à prendre en main. Ex : agent personnalisé pour nos consultants, simplifie la production. | Sécurité optimale : éviter la multiplication des installations et clés d'API. Expertise sur la sécurisation MCP : nous avons déployé un MCP base de connaissance couplé au SSO Azure. |

### Exemple 2 — Dispositif d'accompagnement

| Montée en compétences | Rituels & culture produit | Transfert de compétences | Conduite du changement |
|-----------------------|---------------------------|--------------------------|------------------------|
| Former dans le contexte réel : design system, intégration IA, stratégie design, collaboration pluridisciplinaire. | Installer les boucles de feedback, les design reviews et la culture du test qui rendent les équipes autonomes. | Chaque mission se termine avec des équipes plus compétentes. La documentation survit au départ du consultant. | Accompagner les résistances, embarquer le Trio Stratégique et ancrer les changements dans la durée. |

## Variables / paramètres

* **Titre H2** : énonce le concept global ou la promesse
* **Sous-titre** : optionnel, donne l'angle
* **4 colonnes parfaitement parallèles** :
  * Même structure (titre + description)
  * Descriptions de longueur similaire (équilibre visuel)
  * Pictogrammes optionnels — même style entre les 4
* **Mise en page** : 4 colonnes alignées, gap fluide, hauteur uniforme

## Règles de voix

* **Titres de colonnes** : nominal court (1-3 mots) — "Gouvernance", "Coûts", "Adoption"
* **Descriptions** : phrases courtes (≤ 20 mots), parallélisme syntaxique entre les 4
* **Si verbal** : commencer chaque description par un Nous actif ou un impératif Vous, cohérent entre les 4
* **Vocabulaire** : aligné sur le persona dominant (vocabulaire ROI/risque pour Marc-Antoine, technique pour Alexis…)

## Anti-patterns

* ❌ 5 colonnes (saturation : repenser en 4, ou passer à T07 six pillars)
* ❌ 3 colonnes "déguisées en 4" avec une colonne creuse (préférer T05)
* ❌ Colonnes de longueurs très différentes
* ❌ Mélange de registres entre colonnes (ex: 3 nominales + 1 verbale)
* ❌ Plus de 30 mots par colonne

## Pattern Frontguys récurrent

* **Axes IA opérationnels** : Gouvernance · Coûts · Adoption · Sécurisation
* **Accompagnement** : Montée en compétences · Rituels · Transfert · Conduite du changement
* **4 piliers méthodo** : Diagnostic · Cadrage · Priorisation · Plan d'action
